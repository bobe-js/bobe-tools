1. 采用 begin / complete 模式进行 虚拟代码 构建

2. 每个目标一个上下文，包含 

   1. 头部 class 属性解构，不需要做映射
   2. `bobe<Props>` 类型解构
      1. Props 需要被桥接 类型代替，当原文件查找 Props 位置的引用数量时
         1. 引用位置处于字面量内，则需要到对应虚拟文档中查找
            找到引用 `a:a` 需要二次调用查找 `:a` 位置
         2. 引用位置处于泛型工具 `TypeReference` 时不需要做特殊处理


   ```ts
   bobe<Props>
   
   {
     const { a:a } = {} as Props;
   }
   ```

3. 查找模板中的引用时，

   1. 如果找到的是 `:a` 二次查找
      1. 二次查找，找到仍在虚拟部分，通过 map 找到原字面量的位置
      2. 二次查找，找到在普通的位置，将其作为
   2. 找到的不在 虚拟文档，直接作为引用
